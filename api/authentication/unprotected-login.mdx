<Warning>
This endpoint is intended for development and testing purposes only. It stores the complete user object (including password hash) in the JWT token, unlike the protected `/login` endpoint which uses a DTO. Do not use in production.
</Warning>

This endpoint authenticates a user and returns a JWT token containing the full user object stored in the `unprotectedCookie`.

## Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password (plaintext, will be verified against bcrypt hash)
</ParamField>

## Response

<ResponseField name="status" type="string">
  Status of the operation ("success")
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message ("Unprotected Logged in")
</ResponseField>

The JWT token is stored in a cookie named `unprotectedCookie` with a 1-hour expiration.

### Cookie Details

- **Name**: `unprotectedCookie`
- **Max Age**: 3600000ms (1 hour)
- **Contains**: Full user object including `_id`, `first_name`, `last_name`, `email`, `password`, `role`, `pets`

<CodeGroup>
```bash cURL
curl -X GET http://localhost:8080/api/sessions/unprotectedLogin \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "mypassword"
  }' \
  -c cookies.txt
```

```javascript JavaScript
const response = await fetch('http://localhost:8080/api/sessions/unprotectedLogin', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'mypassword'
  }),
  credentials: 'include'
});

const data = await response.json();
console.log(data);
```
</CodeGroup>

## Response Examples

### Success Response (200)

```json
{
  "status": "success",
  "message": "Unprotected Logged in"
}
```

### Error Responses

**400 Bad Request** - Missing required fields
```json
{
  "status": "error",
  "error": "Incomplete values"
}
```

**404 Not Found** - User doesn't exist
```json
{
  "status": "error",
  "error": "User doesn't exist"
}
```

**400 Bad Request** - Incorrect password
```json
{
  "status": "error",
  "error": "Incorrect password"
}
```

**500 Internal Server Error**
```
Ha ocurrido un error en la petición, por favor ver detalle: [error details]
```

## Differences from `/login`

| Feature | `/login` (Protected) | `/unprotectedLogin` |
|---------|---------------------|---------------------|
| Cookie name | `coderCookie` | `unprotectedCookie` |
| Token content | UserDTO (name, role, email) | Full user object with password |
| Use case | Production | Development/Testing |
| Security | High (minimal data in token) | Low (includes sensitive data) |

## Implementation Details

The endpoint uses the same authentication logic as `/login` but stores the complete user object in the JWT token instead of using a DTO. This can be useful for testing but should never be used in production environments.
