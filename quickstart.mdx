## Prerequisites

Before you begin, ensure you have the following installed:

- Node.js version 22 or higher
- MongoDB (local or cloud instance)
- Git

<Note>
To check your Node.js version, run `node --version` in your terminal. If you need to upgrade, visit [nodejs.org](https://nodejs.org).
</Note>

## Get up and running

<Steps>
  <Step title="Clone the repository">
    Clone the Adoptme API repository to your local machine:
    
    ```bash
    git clone https://github.com/GitRepoFabi/coderhouse-backendIII.git
    cd coderhouse-backendIII
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages:
    
    ```bash
    npm install
    ```
    
    This will install:
    - express - Web framework
    - mongoose - MongoDB object modeling
    - jsonwebtoken - JWT authentication
    - bcrypt - Password hashing
    - dotenv - Environment variable management
    - multer - File upload handling
    - swagger-jsdoc & swagger-ui-express - API documentation
    - @faker-js/faker - Mock data generation
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the root directory with the following variables:
    
    ```bash
    PORT=8080
    MONGO_URL=mongodb://127.0.0.1:27017
    DB_NAME=adoptme
    ```
    
    <Info>
    **Default values**: If you don't provide these variables, the API will use the defaults shown above (from `src/config/config.js:5-8`).
    </Info>
    
    For a cloud MongoDB instance (e.g., MongoDB Atlas), use your connection string:
    
    ```bash
    MONGO_URL=mongodb+srv://username:password@cluster.mongodb.net/
    ```
  </Step>

  <Step title="Start the server">
    Start the development server:
    
    <CodeGroup>
    ```bash Production
    npm start
    ```
    
    ```bash Development (with auto-reload)
    npm run dev
    ```
    </CodeGroup>
    
    You should see:
    ```
    Connection successful MongoDB
    Listening on 8080
    ```
  </Step>
</Steps>

## Make your first API request

Now that your server is running, let's make your first API call.

<Steps>
  <Step title="Register a new user">
    Create a user account using the registration endpoint:
    
    ```bash
    curl -X POST http://localhost:8080/api/sessions/register \
      -H "Content-Type: application/json" \
      -d '{
        "first_name": "John",
        "last_name": "Doe",
        "email": "john.doe@example.com",
        "password": "securePassword123"
      }'
    ```
    
    **Response** (from `src/controllers/sessions.controller.js`):
    ```json
    {
      "status": "success",
      "payload": "66abc123def456789"
    }
    ```
  </Step>

  <Step title="Create a pet">
    Add a pet to the system:
    
    ```bash
    curl -X POST http://localhost:8080/api/pets \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Max",
        "specie": "dog",
        "birthDate": "2020-05-15"
      }'
    ```
    
    **Response** (from `src/controllers/pets.controller.js:5-21`):
    ```json
    {
      "status": "success",
      "payload": {
        "_id": "66def789abc123456",
        "name": "Max",
        "specie": "dog",
        "birthDate": "2020-05-15T00:00:00.000Z",
        "adopted": false,
        "image": ""
      }
    }
    ```
  </Step>

  <Step title="Get all pets">
    Retrieve the list of all pets:
    
    ```bash
    curl http://localhost:8080/api/pets
    ```
    
    **Response** (from `src/controllers/pets.controller.js:5-12`):
    ```json
    {
      "status": "success",
      "payload": [
        {
          "_id": "66def789abc123456",
          "name": "Max",
          "specie": "dog",
          "birthDate": "2020-05-15T00:00:00.000Z",
          "adopted": false
        }
      ]
    }
    ```
  </Step>

  <Step title="Create an adoption">
    Connect a user with a pet:
    
    ```bash
    curl -X POST http://localhost:8080/api/adoptions/{userId}/{petId}
    ```
    
    Replace `{userId}` and `{petId}` with actual IDs from previous responses.
    
    **Response** (from `src/controllers/adoptions.controller.js:23-38`):
    ```json
    {
      "status": "success",
      "message": "Pet adopted"
    }
    ```
    
    <Info>
    The adoption process automatically:
    - Updates the pet's `adopted` status to `true`
    - Adds the pet to the user's `pets` array
    - Creates an adoption record
    - Sets the pet's `owner` field
    </Info>
  </Step>
</Steps>

### Generate mock data

Quickly populate your database with test data:

```bash
curl -X POST "http://localhost:8080/api/mocks/generateData?user=5&pet=10"
```

This creates 5 users and 10 pets using realistic data from Faker.js.

<Tip>
The mock data generator is perfect for testing and development. See the implementation in `src/mocks/mocks.js`.
</Tip>

## Common issues

<AccordionGroup>
  <Accordion title="Connection refused to MongoDB">
    Ensure MongoDB is running:
    ```bash
    # Check MongoDB status
    sudo systemctl status mongod
    
    # Start MongoDB
    sudo systemctl start mongod
    ```
    Or verify your `MONGO_URL` is correct if using a cloud instance.
  </Accordion>

  <Accordion title="Port already in use">
    If port 8080 is occupied, change the `PORT` variable in your `.env` file:
    ```bash
    PORT=3000
    ```
  </Accordion>

  <Accordion title="Incomplete values error">
    Ensure you're sending all required fields:
    - **Users**: `first_name`, `last_name`, `email`, `password` (from `src/dao/models/User.js:5-22`)
    - **Pets**: `name`, `specie`, `birthDate` (from `src/controllers/pets.controller.js:16-17`)
  </Accordion>
</AccordionGroup>
